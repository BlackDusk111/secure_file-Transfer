# secure_file-Transfer
SecureTransfer is a Python desktop application for end-to-end encrypted peer-to-peer file transfer, built as coursework for the ST5062CEM Programming and Algorithm 2 module at Softwarica College of IT and E-Commerce. It uses a hybrid RSA-2048 and AES-256-CBC encryption scheme where a fresh random key and initialisation vector are generated for every file sent, ensuring the server can route packets without ever being able to read their contents, and implements SHA-256 file integrity verification so the recipient can confirm no tampering occurred in transit. User accounts are stored with SHA-256 hashed passwords in a local JSON file, and the system includes a dedicated attack detection module covering three threat vectors simultaneously — a sliding-window rate limiter that blocks IP addresses exceeding ten requests per ten seconds as a Denial of Service defence, a nonce-and-timestamp validator that rejects duplicate or stale packets as a replay attack defence, and a public key fingerprinting system that alerts when a returning user's RSA key changes as a Man-in-the-Middle defence. The multi-threaded TCP server handles any number of simultaneous users, each on their own thread, and the Tkinter GUI provides a file hub with an activity log, transfer history table, and a live Attack Monitor panel showing real-time event counts and logs pulled from the server. To run it, install the single dependency with pip install cryptography==42.0.5, start the server with python server.py, then open one or more client windows with python client.py and register any username you like. Run the full test suite with python -m unittest test_securetransfer -v.
